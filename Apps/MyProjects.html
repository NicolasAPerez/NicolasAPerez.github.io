PortfoliOS (Current Website)

Tech Stack

  • JavaScript
  • HTML
  • CSS

Description

This is a website made to mimic a traditional Operating System UI, including applications being opened in movable windows. This was handled through a variety of custom elements to handle the logic of the Shortcuts, Windows, and Taskbar Buttons. The website was revamped in the latter part of 2024 to make the code easier to modify for use outside my own through a better implementation of encapsulation of the underlying logic and additional features such as Alerts that might be useful. Now to use the code with your own website you just need to apply context to the mockOS object found in "WindowManager.js", and have the custom elements included that are found in "CustomElements.js"
Required Context:
  • Where the Custom Elements will be applied (Applications, Shortcut Buttons, and Taskbar Containers)
  • Where the App Data files can be found (Image and App File locations)
  • Insert the Apps themselves (With their icons, data, and names linked)
Originally, the revamp would move the code to be mostly React.js based through Next.js as I was becoming highly familiar with it at the time. However, after some testing and examination, I found that React would not be the best or most efficient solution for this website. Considering this is a Static site, React components would have been overkill for what I needed, and the Static Site Generation of Next.js did not work as well as I hoped.
On a personal note, I've always been nostalgic for Windows XP's user interface and design and the original version of this website tried to emulate that to some degree. After the revamp, I found the design language of 2000's era MacOS to be a big inspiration, with the "shiny" gradients on some elements and some more "matte" gradients on moving elements such as the windows.
